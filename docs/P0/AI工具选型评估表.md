# AI工具选型评估表
| 工具类型 | 评估对象 | 是否选用 | 理由 |
| -- | -- | -- | -- |
| 编程助手 | 通义灵码 | 是 | 深度集成与方便使用，可以直接安装在IDE里，符合日常编程习惯，且本地化支持较好，适用于国内网络环境 |
| 编程助手 | Claude Code | 是 | 自主性极强，且具有顶尖的模型能力，并且全流程覆盖，可以真正参与进开发流 |
| 对话工具 | ChatGPT | 是 | 全能且成熟，尤其对于基础算法的编写能力较强，可以用来辅助编写部分核心代码 |
| 对话工具 | Grok | 是 | 推理能力较强，且具有全量实时数据，尤其擅长文档处理工作 |
| 对话工具 | Deepseek | 是 | 算法与数学能力较强，推理透明化，具有超长 上下文功能，适于项目开发 |
| 代码审查 | CodeRabbit | 是 | 上下文感知可以理解业务逻辑，指出运行时的潜在bug |
| 代码审查 | DeepSource | 否 | 灵活性略低，侧重于规则检查，不像 CodeRabbit那样能进行自由的逻辑探讨 |
| 测试工具 | Testim | 是 | 拥有自我修复能力，且对代码能力要求不高 |
| 测试工具 | Applitools | 否 | 功能局限，对底层接口和复杂业务逻辑的测试较弱 |
